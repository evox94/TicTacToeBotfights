Tic-tac-toe

Game Description:

Tic-tac-toe (also known as noughts and crosses or Xs and Os) is a paper-and-pencil game for two players, X and O, who take turns marking the spaces in a 3x3 grid. The player who succeeds in placing three of their marks in a horizontal, vertical, or diagonal row wins the game.

The following example game is won by the first player, X:

Because of the simplicity of tic-tac-toe, it is often used as a pedagogical tool for teaching the concepts of good sportsmanship and the branch of artificial intelligence that deals with the searching of game trees.

It is straightforward to write a computer program to play tic-tac-toe perfectly, to enumerate the 765 essentially different positions (the state space complexity), or the 26,830 possible games up to rotations and reflections (the game tree complexity) on this space.

How to 'play':

Game state is represented by a 3x3 integer matrix. Integers range from 0 to 2:

  • 0 - empty space
  • 1 - You
  • 2 - Your opponent

This is little bit different then in a more 'traditional' implementation where 1s are Xs and 2s are Os. In this case '1' will ALWAYS be your symbol whether you are an X or an O. This is done so player algorithms are symmetrical and can be interchanged easily.

You can still figure out if you are one or the other, based on the state of the board during your first move(it really shouldn't matter much).

Algorithm example:

package com.example.mypackage;

public class TicTacToeBot {

    final public static int FREE_SPACE = 0;
    final public static int MY_SYMBOL = 1; //You are always 1
    final public static int OPPONENT = 2;

    public static void move(int[][] board){
        //your code goes here:
        //example code:
        //play on the first 'free' board square
        for (int i = 0; i < 3; i++) {
            for (int j = 0; j < 3; j++) {
                if(board[i][j] == FREE_SPACE){
                    //You output your move by using System.out.println() or System.out.print()
                    //If you have a need for debuging use System.err
                    System.out.println(i+" "+j);//this is one of the formats game accepts
                    return;                     //be sure to return unless you want to output more moves;
                }
            }
        }
    }
}

When you are ready to output your move you can do that by calling System.out.println() or System.out.print() methods.
Here are the string formats that you can use to output your move:

  • 0 2
  • 0,2
  • 0, 2
  • 02

All of these mean that you want to play your symbol(X or O) at the square: row '0' column '2'. That's the right most upper corner of the board. If you output for example System.out.println("1 1") that means that you want to play at the middle of the board.

Each call to the System.out.println() is interpreted as one move, so be sure to use it once per method call, unless you can predict the future.

Any invalid move (out-of-bounds/playing on a filled square) will stop the game and you will lose.

If you have a need to debug use System.err. You will be able to see the output in both the System and Simulator consoles.

There are no early draws in the game, game keeps going on until there is a clear winner or until the 3x3 grid is filled. If there isn't a winner after the grid is filled, a draw is called.
